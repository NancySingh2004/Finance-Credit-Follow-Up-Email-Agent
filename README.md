
# AI Finance Email Agent

An AI-powered Finance Follow-Up Agent built using Streamlit, LangChain, and Groq LLM that automates invoice reminder workflows with intelligent tone escalation, dry-run email testing, and audit logging.

---

# Project Overview

The AI Finance Email Agent helps finance teams automate pending payment reminder workflows.

The system:

* Reads invoice data from CSV
* Detects overdue invoices
* Applies tone escalation logic
* Generates AI-powered payment reminder emails
* Supports dry-run testing mode
* Maintains audit logs
* Flags invoices requiring manual/legal review

---

# Features

## Intelligent Tone Escalation

| Days Overdue | Tone             |
| ------------ | ---------------- |
| 1–7 Days     | Warm & Friendly  |
| 8–14 Days    | Polite but Firm  |
| 15–21 Days   | Formal & Serious |
| 22–30 Days   | Stern & Urgent   |
| 30+ Days     | Legal Escalation |

---

## AI Email Generation

Uses Groq + Llama 3.1 to generate:

* Professional payment reminders
* Dynamic escalation tone
* Context-aware finance communication

---

## Dry Run Mode

Prevents accidental real-world emails during testing.

```python
DRY_RUN = True
```

When enabled:

* Emails are printed in terminal
* No actual SMTP email is sent

# Prompt Design

## System Prompt Strategy

The prompt enforces:

* Professional finance tone
* No placeholders
* No extra explanations
* Dynamic escalation behaviour

### Example Prompt

```python
template = """
You are a professional finance assistant.

Write a clear and professional payment reminder email.

STRICT RULES:
- No placeholders
- No extra explanations
- Tone must match: {stage}
- Output ONLY the email
"""
```

# Security Risk Mitigation

## Prompt Injection

### Risk

Malicious invoice data manipulating LLM behaviour.

### Mitigation

* Strict prompt instructions
* Controlled input variables
* Structured prompt design

---

## Data Privacy / PII

### Risk

Invoice data contains personal information.

### Mitigation

* Local CSV processing
* Minimal data sent to LLM
* No external database exposure

---

## API Key Exposure

### Risk

Leaking Groq or Gmail credentials.

### Mitigation

* `.env` usage
* `python-dotenv`
* `.gitignore` protection

Example:

```env
GROQ_API_KEY=your_key
GMAIL_USER=your_email
GMAIL_APP_PASSWORD=your_password
```

---

## Hallucination Risk

### Risk

Incorrect finance communication generated by LLM.

### Mitigation

* Human review before dispatch
* Editable email drafts
* Dry-run verification

---

## Unauthorized Access

### Risk

Unauthorized triggering of workflows.

### Mitigation

(Current Prototype)

* Localhost-only execution

Future Enhancements:

* OAuth authentication
* JWT tokens
* Role-based access

---

## Email Spoofing

### Risk

Emails appearing from unverified senders.

### Mitigation

* Gmail SMTP verified sender
* Dry-run mode during development
